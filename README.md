# ARKitRecorder
Record image and camera pos, etc. information provided by ARKit and save them into files.

## Demo
![gif](https://raw.githubusercontent.com/StephenGuanqi/AVResources/arkitrecorder/ARRecorder.gif)

The app will start recording the ARKit Info when users has pressed down the button. Every time the  the `ARFrame` has been updated, the session update delegate method is called. We then extract the `CVPixelBuffer` from the `ARFrame` and convert it into jpg, and write to local file. We also convert other information including `ARCamera`, `LightEstimate` and `PointCloud `information into json object and stores it into the member variable. You can find the json format in the following section.

When the button is released, all of the json object will be written into the local json file, and the objects will be removed and released. The images and json file will be written into the folder into app's document directory, and the name of the folder will be the timestamp when the button is pressed, a.k.a. the time when the session starts recording.

All of the process above is done asyncronously.

## Repository Content

The repository has two project folders. The `ARRecorder` is implemented in swift and the `OCARKitTest` is implemented in objective-C. They have the same function as discussed above, except that the objective-C project couldn't store the point cloud info. I don't know have to get it from a `Vector3*` in the ARFrame. If you do know, please make an issue or create a pull request.
